# TheReplyte — WhatsApp AI Agent

TheReplyte is a no-code WhatsApp AI agent platform. Businesses get an AI that answers customer messages from their own knowledge base, hands off to human agents in Chatwoot when needed, and reports back to the owner. The TheReplyte dashboard is used by platform staff only, for client onboarding and AI configuration; the client's agents work entirely inside Chatwoot.

- Dashboard: `https://app.thereplyte.com`
- API: `https://api.thereplyte.com`
- Agent workspace: Chatwoot at `https://inbox.thereplyte.com`

---

## Architecture

```
End Customer (WhatsApp)
        ↓
Meta WhatsApp Cloud API
        ↓
TheReplyte API (api.thereplyte.com)        ← apps/whatsapp-agent-api (NestJS + Prisma + Postgres/pgvector)
  ├─ AI replies from knowledge base (Kimi + RAG)
  ├─ Media understanding (voice/image/documents)
  ├─ Human handoff to Chatwoot (summary, labels, CSAT)
  └─ Forwards agent replies back to WhatsApp
        ↓
Chatwoot (inbox.thereplyte.com)            ← infra/chatwoot
        ↓
Client's agents reply

TheReplyte Staff
        ↓
TheReplyte Dashboard (app.thereplyte.com)  ← apps/whatsapp-agent-web (Next.js 14)
  ├─ Clients (onboarding)
  ├─ AI Settings (per client)
  ├─ Knowledge Base (per client)
  ├─ Canned Responses
  ├─ Analytics
  └─ Playground
```

Supporting services: Postgres (with `pgvector`), Chatwoot, n8n (optional alerts), Traefik (routing/TLS). See `infra/whatsapp-agent-api/` and `infra/whatsapp-agent-web/` for the Docker Compose services.

## Features (current)

### Onboarding (no-code)
- Staff creates a client in the dashboard; the system automatically creates a dedicated Chatwoot account, API inbox, and Chatwoot webhook; creates a default client admin user and auto-confirms it (direct DB update via `CHATWOOT_DATABASE_URL`, so SMTP is not required); stores Meta credentials; applies default AI settings.
- Client card shows Chatwoot connection status, Meta webhook status (green when a webhook was received in the last 24h), and quick actions: Open Chatwoot, Meta Setup, Refresh Status, Edit AI Settings.
- Meta Setup panel shows the callback URL, verify token, and step-by-step instructions (guided copy-paste flow).

### AI message handling
- Replies generated by Kimi using the client's system prompt, temperature, and max tokens.
- Knowledge base RAG: pgvector similarity search injects relevant chunks into the prompt; answers are constrained to the KB + business info.
- Confidence threshold: if the best KB chunk similarity is below the client's threshold, the AI uses the fallback message and hands off.
- Handoff keywords (per client, with built-in defaults) trigger immediate human handoff.
- Operating hours (per client, timezone-aware, overnight ranges, closed days): outside-hours messages get the outside-hours reply and are queued for a human.
- Welcome message on a customer's first-ever conversation.
- Multi-language: the AI detects the customer's language and replies in it, falling back to the client's business language.
- Media: voice notes transcribed (Whisper), images described with caption extraction (Kimi vision), PDF/DOCX/TXT text extracted; unprocessable media becomes a placeholder note.

### Human handoff & Chatwoot
- Handoff creates/links the Chatwoot contact and conversation and pushes the full bot/customer transcript.
- An AI-generated conversation summary is saved on the conversation and posted as a Chatwoot private note.
- Labels applied automatically: rule-based (`ai-handoff`, `after-hours`/`in-hours`, `urgent`) plus 1–3 AI-suggested topic labels.
- Agent replies in Chatwoot are forwarded to the customer's WhatsApp automatically.
- Canned responses: per-client templates; agents type `/shortcut` in Chatwoot and the webhook expands `{{customer_name}}`, `{{business_name}}`, `{{agent_name}}` before sending.

### CSAT
- When a conversation is resolved in Chatwoot, the customer receives a 1–5 rating request (toggleable per client).
- The next numeric reply is stored (`csatRating`/`csatFeedback`), synced to the handoff log, thanked, and posted as a Chatwoot private note. Any non-rating reply expires the request.

### Analytics
- `GET /api/analytics/overview?clientId=` returns: conversation counts, handoffs, today's activity, top handoff reasons, 7-day daily volume, average resolution time, average handoff response time, CSAT (average + responses), AI token usage (prompt/completion/total), and estimated AI cost in USD (from `AI_INPUT_PRICE_PER_1M_TOKENS` / `AI_OUTPUT_PRICE_PER_1M_TOKENS`).
- Dashboard Analytics page with client selector and stat cards.

### AI testing playground
- Dashboard page + `POST /api/playground/:clientId/message`: runs the real pipeline (client settings + KB search + Kimi) read-only — no WhatsApp sends, no conversation records, no handoffs.
- Returns the reply, handoff recommendation + reason, token usage, and the KB sources used (with similarity scores).

### Knowledge base
- Per-client documents with pgvector embeddings.
- Add plain text entries, upload `.txt`/`.pdf`/`.docx` files, or crawl a public website URL (HTML stripped to text, chunked, embedded).

## Dashboard pages

| Page | Purpose |
|---|---|
| `/dashboard/clients` | Client list, onboarding, Chatwoot/Meta status, quick actions |
| `/dashboard/ai-settings` | Per-client AI behaviour (prompt, model, temperature, max tokens, confidence threshold, messages, handoff keywords, operating hours, CSAT) |
| `/dashboard/knowledge-base` | Text entries, file upload, website crawl, document list |
| `/dashboard/canned-responses` | `/shortcut` templates with variables |
| `/dashboard/analytics` | Usage, handoff, CSAT, token and cost metrics |
| `/dashboard/playground` | Test the AI against a client without sending messages |

Note: there is intentionally **no conversations page** — platform staff do not view client conversations; those live in each client's Chatwoot account.

## Tech stack

- **API** `apps/whatsapp-agent-api` — NestJS 10, Prisma 5, PostgreSQL + pgvector, JWT auth, class-validator.
- **Dashboard** `apps/whatsapp-agent-web` — Next.js 14 (app router), plain inline-style components, token stored in `localStorage`.
- **AI** — Kimi (Moonshot) chat + vision, OpenAI-compatible embeddings, Whisper-compatible transcription.
- **Infra** — Docker Compose per app under `infra/`, Traefik reverse proxy, shared `cloudit` network, external Postgres service.

### Main API modules

`auth`, `clients`, `chatwoot` (webhook + API client), `whatsapp` (webhook + message pipeline), `conversations`, `ai`, `knowledge-base`, `canned-responses`, `analytics`, `playground`, `media`.

### Key data models

`Client` (onboarding + all AI settings), `Customer`, `Conversation` (status, CSAT fields, `summary`, Chatwoot links), `Message` (with `kimiMetadata` token usage), `HandoffLog` (reason, response time, satisfaction), `Document` (KB chunks + embeddings), `CannedResponse`, dashboard `User`.

## Environment variables (API)

See `apps/whatsapp-agent-api/.env.example` for the full commented list. Highlights:

- `DATABASE_URL` — Postgres (pgvector-enabled).
- `JWT_SECRET`, `JWT_EXPIRES_IN` — dashboard auth.
- `META_API_BASE_URL`, `META_VERIFY_TOKEN`, `META_WEBHOOK_SECRET` — WhatsApp Cloud API.
- `KIMI_API_KEY`, `KIMI_API_URL`, `KIMI_MODEL`, `KIMI_VISION_MODEL` — AI replies + image understanding.
- `EMBEDDING_API_URL`, `EMBEDDING_API_KEY`, `EMBEDDING_MODEL` — KB embeddings.
- `WHISPER_API_URL`, `WHISPER_API_KEY`, `WHISPER_MODEL` — voice-note transcription (falls back to `EMBEDDING_API_KEY`).
- `CHATWOOT_PLATFORM_API_URL`, `CHATWOOT_PLATFORM_API_KEY`, `CHATWOOT_ADMIN_API_KEY`, `CHATWOOT_ADMIN_USER_ID`, `CHATWOOT_WEBHOOK_SECRET` — Chatwoot integration.
- `CHATWOOT_DATABASE_URL` — direct DB access to auto-confirm Chatwoot users (skips SMTP dependency) and to apply installation branding.
- `CHATWOOT_BRAND_NAME`, `CHATWOOT_BRAND_URL`, `CHATWOOT_BRAND_LOGO_URL` — TheReplyte branding written to Chatwoot's `installation_configs` on API start (instance-wide, all accounts). Defaults: `TheReplyte` / `https://thereplyte.com` / no logo. Restart `chatwoot-rails` once after first apply (Chatwoot caches these configs).
- `API_PUBLIC_URL` — used when registering the Chatwoot webhook.
- `AI_INPUT_PRICE_PER_1M_TOKENS`, `AI_OUTPUT_PRICE_PER_1M_TOKENS` — cost estimate (defaults 0).
- `N8N_WEBHOOK_URL` — optional urgent-handoff alerts.
- `CORS_ORIGIN` — dashboard origins.

Per-client Meta credentials (access token, phone number ID, verify token) are stored on the `Client` record, not in env.

## Local development

```bash
# API
cd apps/whatsapp-agent-api
cp .env.example .env          # fill in values
npm run db:generate
npm run db:migrate
npm run start:dev             # http://localhost:3010

# Dashboard
cd apps/whatsapp-agent-web
npm run dev                   # http://localhost:3011
```

Production database changes: create a migration folder under `apps/whatsapp-agent-api/prisma/migrations/` and apply with `npm run db:deploy` on the server.

## Deployment

Server path: `/opt/cloudit/cloudit-platform` (see `docs/deployment-guide.md`).

```bash
cd /opt/cloudit/cloudit-platform && git pull

cd infra/whatsapp-agent-api && docker compose up -d --build   # api.thereplyte.com
cd ../whatsapp-agent-web && docker compose up -d --build      # app.thereplyte.com
```

Env changes to the API require `docker compose up -d --force-recreate` (a plain restart does not re-read `env_file`).
